Penova 1.2.0

2 May 2026 · macOS 14+ · iOS 17+

Fountain — Penova's portable interchange format

  • Penova now treats .fountain plain text as a canonical interchange format. Every Project field that has a Fountain representation round-trips losslessly through the parser and exporter — including title-page metadata, episode boundaries, scene beats, locked-script state, and revision history. Full dialect spec documents the contract.
  • The parser handles all six standard fountain.io title-page keys plus Penova-namespaced extensions; it gracefully tolerates unknown Penova-* keys (forward-compat for future versions). FADE OUT:, IRIS OUT, and SMASH CUT TO now recognize as transitions.
  • End-to-end validated against three canonical reference screenplays (Big Fish, Brick & Steel, The Last Birthday Card) — they round-trip through the full Penova pipeline with byte-equivalent structure.

iOS — title-page editor + scene-creation flow

  • Title page editor on iPhone now auto-resizes for the keyboard — drag the form down to dismiss, or tap the keyboard's "Done" button. Multi-line fields (Contact, Notes) no longer hide behind the keyboard on small screens.
  • The keyboard accessory's "+ New Scene" button now opens the proper New Scene sheet so you fill in heading + location + time-of-day instead of being left with an orphaned "NEW LOCATION - DAY" placeholder.

Foundation — SwiftData migration plan

  • Penova now declares an explicit SwiftData VersionedSchema (SchemaV1) and SchemaMigrationPlan (PenovaMigrationPlan). Today the plan is a no-op — every property change through 1.x has been an Optional-with-default that SwiftData migrates lightweight automatically. The scaffolding exists so the first time we rename or remove a model property, the migration path is already in place — no scrambling to declare a versioned schema after a returning user has hit a launch crash.

Accessibility

  • Cross-platform A11yID identifiers consolidated into PenovaKit; iOS surfaces (Settings, Reports, Project Detail, Scene Detail, Title Page) now expose stable identifiers for UI tests + accessibility tools.

Tests

  • 434 / 434 unit tests across 45 suites pass. New: PenovaMigrationPlanTests pins the schema version, the V1 model list, and that ModelContainer(migrationPlan:) initializes cleanly.
