Penova 1.1.2

2 May 2026 · macOS 14+

Visible polish

  • Element-kind chip ("ACTION" / "TRANSITION" / etc.) now appears only on the focused row. Unfocused rows render like an actual screenplay — transitions right-aligned ALL CAPS, dialogue indented, no inline editor widgets. The chip stays available on the focused row so you can tap to cycle the row's kind.
  • Indent ladder fix — the iOS editor's character / dialogue / parenthetical columns now sit at the WGA-correct positions on unfocused rows (was shifted ~60pt left because the badge slot was being subtracted even when nothing was rendering there).
  • Default Mac window size bumped to 1280×820 so the 640pt paper page fits naturally between the library sidebar and the scene inspector — no more horizontal scrolling on first launch. Paper now also adapts gracefully when you drag the window narrower.
  • Empty-scene flow — tapping "Start writing" on a brand-new scene previously inserted a duplicate scene-heading row. Now starts you on an empty action row, which is what the next thing you'd type was anyway.

Editor stability

  • Mac line-delete renumbering — repeated insert-above + delete sequences could leave duplicate order values among scene elements; ForEach then reordered rows non-deterministically between reloads. Delete now compacts orders to a contiguous 0..N-1 sequence so this can't happen.

App Store readiness

  • Both targets now declare LSApplicationCategoryType: public.app-category.productivity — App Store Connect required for submission, and Finder shows the right category in Get Info.
  • New PrivacyInfo.xcprivacy manifest on the Mac target (mirrors the iOS one). Required for any future Mac App Store submission and good hygiene for direct-distribution.
  • SIL Open Font License headers (Inter, Roboto Mono, Playfair Display) now ship with the Mac bundle alongside the iOS one.
