Penova 1.1.0

2 May 2026 · macOS 14+

What's new

  • Send Feedback… — new menu item under Penova → and Help. Composes a prefilled email straight to the developer with your app + macOS version, locale, and a UTC timestamp. No analytics; nothing leaves your Mac without you pressing Send.
  • Title page polish — the editor now persists every WGA-format field (title, written by, author, source, contact, draft date, draft stage, copyright, notes). The PDF cover sheet renders the full layout with revision history stacked above the contact block on production drafts.
  • Per-page revision color stripe — locked production drafts now get the WGA-correct three indicators on every page that contains a changed line: a coloured stripe in the right margin, an italic header slug ("Blue Revision — date — TITLE"), and a * in the right gutter beside the changed paragraph (with the standard 3-line consolidation rule for dialogue blocks).
  • Extended revision palette — 20 colors covering the full WGA cycle (white through cherry, then tan, ivory, double-white through double-cherry).
  • "Start New Revision…" command — ⌥⌘R in the Production menu cycles through the WGA color sequence and snapshots your script's current Fountain content for restore.

Round-trip improvements

  • Final Draft (.fdx) export now emits a proper <TitlePage> block that other Final Draft installs read cleanly.
  • Fountain export emits all six standard fountain.io title-page keys (Title, Credit, Author, Source, Draft date, Contact) with multi-line continuation indents; multi-line contact blocks now round-trip through Fountain.
  • FDX import recovers the title page from header/footer text runs in addition to the legacy colon-prefixed form.

Under the hood

  • SwiftData migration is automatic — open a v1.0.0 project in 1.1.0 and the title-page metadata hydrates from your existing project title + contact block. No manual steps.
  • Edits made while a revision is active are stamped against that revision so the page-stripe + asterisks know what to mark.
  • Notarized by Apple. Signed with a Developer ID Application certificate. Auto-update verified by EdDSA signature against the public key in the bundled Info.plist.

Update path

Penova checks for updates automatically once a day via Sparkle. Force a check anytime via Penova → Check for Updates….
